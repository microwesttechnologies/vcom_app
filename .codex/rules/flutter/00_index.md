## Regla de interpretacion
- `MUST`: obligatorio, no negociable.
- `SHOULD`: recomendado fuerte; si no se cumple debe documentarse el motivo.
- `MUST NOT`: prohibido.

## Prioridad cuando hay conflicto
1. Correctitud funcional.
2. Seguridad y estabilidad.
3. Performance medible.
4. Legibilidad y mantenibilidad.
